# Documentação Do Projeto

A pasta `docs/` é a fonte da verdade da landing page do SBF / Escola Regional de Física.

Use estes documentos para orientar planejamento, implementação e validação:

- `requisitos.md`: define objetivo, escopo, requisitos, restrições e perguntas abertas.
- `backlog.md`: tabela central do backlog, com links para Epics e User Stories.
- `backlog/`: arquivos detalhados por Epic, cada um contendo suas User Stories e Tasks.
- `cronograma-4-semanas.md`: plano de execução do projeto do site entre 22/04/2026 e 22/05/2026.
- `integracoes.md`: integrações do projeto, incluindo notificações GitHub para Slack.

## Regras De Manutenção

- Atualize `requisitos.md` quando houver mudança de escopo, público, conteúdo obrigatório, restrição ou critério de sucesso.
- Atualize `backlog.md` sempre que um Epic ou uma User Story for criado, removido, renomeado, priorizado, iniciado, concluído ou bloqueado.
- Atualize o arquivo correspondente em `backlog/` sempre que critérios de aceitação, dependências, Tasks ou validações forem alterados.
- Registre decisões e dúvidas no documento mais próximo do tema.
- Se uma informação estiver fora de `docs/`, trate como contexto auxiliar até ser consolidada aqui.
