# EPIC 8: Comunicação E Divulgação Até Setembro

Objetivo:
Usar o site como base oficial para divulgar o evento e manter informações atualizadas até sua realização.

Publico-alvo:
Participantes potenciais, equipe de comunicação, comissão organizadora e instituições parceiras.

Escopo inicial:
- Plano de divulgação.
- Canais de comunicação.
- Calendário de atualizações.
- Rotina de manutenção do site.

Fora de escopo:
- Operação completa de redes sociais, salvo planejamento básico.

Métricas de sucesso:
- Canais de divulgação definidos.
- Site atualizado conforme decisões do evento.
- Participantes encontram informação confiável até setembro.

Riscos e dependências:
- Falta de responsáveis por atualização.
- Mudanças frequentes de programação ou inscrição.

Status:
Em refinamento

## US 8.1: Planejar Comunicação Do Evento

Como organização,
quero ter um plano de divulgação,
para atrair participantes até setembro.

Contexto:
A landing page será a base da comunicação, mas precisa estar conectada a canais de divulgação.

Critérios de aceitação:
- Dado que o site está publicado, quando a divulgação começa, então há canais definidos para distribuir o link.
- Dado que há datas importantes, quando o calendário é criado, então comunicações relevantes são planejadas.
- Dado que existem responsáveis, quando uma comunicação precisa sair, então a equipe sabe quem executa.

Prioridade:
Média

Estimativa:
M

Dependências:
- Site publicado.
- Definição de responsáveis por comunicação.

Status:
A fazer

### Tasks Da US 8.1

| Task | Resultado esperado | Status |
| --- | --- | --- |
| TASK 8.1.1: Definir canais de divulgação | Canais de comunicação definidos. | A fazer |
| TASK 8.1.2: Criar calendário de divulgação | Calendário macro de comunicação até setembro. | A fazer |

Validação:
- Lista de canais aprovada pela organização.
- Calendário revisado com equipe de comunicação.

## US 8.2: Manter Site Atualizado Até O Evento

Como participante,
quero encontrar informações atualizadas,
para confiar no site como fonte oficial do evento.

Contexto:
Após o MVP, o site deve acompanhar definições de programação, inscrições e comunicação até setembro.

Critérios de aceitação:
- Dado que uma informação oficial muda, quando o site é revisado, então o conteúdo é atualizado.
- Dado que a programação é confirmada, quando o usuário acessa o site, então encontra a versão mais recente.
- Dado que há informação pendente, quando ela ainda não foi decidida, então o site evita comunicar como definitivo.

Prioridade:
Média

Estimativa:
M

Dependências:
- Responsável por atualização.
- Fluxo de aprovação de conteúdo.

Status:
A fazer

### Tasks Da US 8.2

| Task | Resultado esperado | Status |
| --- | --- | --- |
| TASK 8.2.1: Definir responsável por atualizações | Responsável ou grupo responsável pela manutenção do site definido. | A fazer |
| TASK 8.2.2: Criar rotina de revisão do site | Rotina de revisão quinzenal ou mensal definida até setembro. | A fazer |

Validação:
- Responsável comunicado à equipe.
- Rotina registrada no planejamento do projeto.
